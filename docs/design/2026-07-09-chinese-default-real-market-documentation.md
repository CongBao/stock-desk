# 中文默认与真实行情文档设计

## 背景

stock-desk v1 面向中文 A 股个人用户。当前仓库默认展示英文 README，Wiki 默认首页也是英文；现有公开截图使用可识别的合成波形行情，并包含 `Synthetic`、`CC0 Demo`、Worker 未检测或阶段版本标签。这些内容能够支持自动化测试，却不适合作为正式产品介绍或低代码用户教程。

本设计将公开文档改为简体中文默认、英文可切换，并使用产品真实运行界面和真实 A 股历史数据重新拍摄文档截图。原始行情、用户凭据、私有数据库和拍摄环境不进入公开仓库。

## 目标

1. GitHub 仓库和 Wiki 的默认入口均为简体中文，并能一键切换英文。
2. README 保持简洁，只承担产品介绍、核心能力、代表性截图、普通用户安装入口和 Wiki 导航。
3. 每项已交付功能都能从双语功能索引进入详细使用页。
4. 行情、公式和回测截图使用真实 A 股历史数据，不出现明显的合成行情。
5. 普通用户可以按 Wiki 完成 Windows 或 macOS 安装、首次启动和核心工作流，不需要源码构建或先学习 GitHub CLI。
6. 截图具备可审计的来源、版本、路由、viewport、哈希和脱敏记录。

## 非目标

- 不把真实行情快照内置到发行包或 Git 仓库。
- 不改变产品的数据源优先级、回测语义或分析逻辑。
- 不把截图中的历史表现解释为投资建议、收益承诺或个股推荐。
- 不为文档截图新增绕过数据许可、凭据保护或来源追溯的产品接口。
- 不要求产品 UI 在本次工作中新增英文界面；英文 Wiki 可以使用中文产品截图并提供英文编号说明。

## 语言与入口结构

### README

- `README.md` 是简体中文默认入口，首行链接 `README.en.md`。
- `README.en.md` 是英文入口，首行链接回 `README.md`。
- 删除旧的简体中文别名文件，只保留两份权威内容，避免三份文档漂移。
- 两份 README 保持相同章节顺序、截图和安装入口。
- README 只保留：产品定位、四项核心能力、4–6 张代表性截图、Windows/macOS 下载入口、Wiki 入口、安全与非投资建议提示。
- Docker、源码贡献、attestation 命令和详细排障移至 Wiki 或维护者文档。普通安装流程不以 GitHub CLI 为前置条件。

### GitHub Wiki

- `Home.md` 是中文默认首页，`Home-en.md` 是英文首页。
- 中文功能页使用无语言后缀的稳定名称；英文配对页使用 `-en` 后缀。
- 每页顶部提供语言切换，底部提供上一页、下一页、功能索引和首页链接。
- `_Sidebar.md` 提供中文共享目录，`_Sidebar-en.md` 提供英文共享目录；若 GitHub Wiki 只能渲染一个侧边栏，则共享侧边栏同时列出中文与 English 入口，页面内部导航仍保持语言一致。
- 旧的 `-zh-CN` 页面在发布前移除；所有仓库和 Wiki 内链一次性更新，避免长期维护重复别名页。

## 真实行情拍摄环境

### 隔离边界

拍摄使用独立的临时应用数据目录和数据库。数据通过 stock-desk 现有适配器进入统一行情模型，并保留产品自身的来源、复权、数据版本和路由证据。拍摄完成后删除临时数据库、原始下载、日志和浏览器 trace。

公开提交只包含：

- PNG/WebP 产品截图；
- Markdown 图注；
- 一份不含原始行情的截图清单；
- 自动验证所需的公开元数据和文件哈希。

公开提交不得包含 Token、API Key、通达信个人目录、数据库、Parquet、CSV、备份、模型提示或可批量还原的授权行情。

### 数据源和截止日

拍摄优先使用产品已支持且允许当前用途的 BaoStock、Tushare 或 AKShare 适配器。每张行情相关截图记录实际命中的来源和最近完成交易日。若主来源不可用或许可边界不清晰，则更换来源或停止该截图，不使用测试夹具冒充真实数据。

同一工作流中的公式预览和回测必须固定相同的股票、周期、复权方式、日期区间和数据版本。数据截止日使用拍摄时可验证的最近完成交易日，不使用未来时间或拍摄时尚未收盘的数据。

### 股票选择

不同功能页可以使用最适合说明该功能的真实股票。候选包括但不限于：

- 贵州茅台：长期日线和复权图；
- 宁德时代：MACD 等趋势/震荡信号示例；
- 平安银行：单股回测与交易明细；
- 招商银行：分析报告和证据链。

拍摄前以图形可读性、数据完整性和足够的非退化信号数量选择股票，不根据回测收益挑选“最好看”的结果。页面和图注必须显示公司名与代码，并统一标注“仅作功能演示，不构成投资建议”。

## 截图规范

1. 正常流程截图显示 `v1.0.0` 或当前补丁版本、API 正常和 Worker 正常。
2. 正常流程不得出现 `Synthetic`、`CC0 Demo`、Worker 未检测、无关失败或无关取消任务。
3. 排障页面允许展示失败，但必须同时展示错误原因、错误码或恢复操作。
4. 每个独立界面状态至少一张图；向导、安装、回放等多步骤流程使用 2–4 张图。
5. 安装文档包含 Windows 安装器、macOS DMG、首次启动和可能出现的系统安全提示。
6. 响应式文档至少包含桌面和窄屏两种 viewport，证明侧栏自动收回、图标缩略和组件无重叠。
7. 英文页面可复用中文 UI 截图，但必须用英文图注和编号明确对应控件。
8. 截图保持原始产品像素，不通过图像生成或绘制改变 K 线、指标、收益或产品状态。允许无损裁剪和遮盖非产品的操作系统个人信息，但遮盖动作必须记录。

## 截图清单

每张截图记录以下字段：

- 文件名和 SHA-256；
- 中文页、英文页和功能 ID；
- 页面路由和界面状态；
- 股票名称、代码、周期、复权和日期范围；
- 数据来源、数据截止日和产品数据版本；
- Git tag、commit 和产品版本；
- viewport 和设备像素比；
- 捕获方式；
- 凭据、路径、日志和个人信息脱敏结果；
- 是否存在裁剪或遮盖；
- 审阅状态。

不涉及行情的安装、设置和运维截图将股票字段标记为“不适用”，但仍记录版本、路由/操作系统状态、viewport、哈希和脱敏结果。

## 捕获流程

1. 创建隔离分支和临时应用数据目录。
2. 启动 API、Worker 和 Web，确认版本与健康状态。
3. 通过现有设置和更新流程导入真实数据，核对来源、截止日和复权。
4. 创建公式版本，并在固定数据快照上完成预览与回测。
5. 使用浏览器逐路由完成操作和视觉检查。
6. 通过 Playwright 或内嵌浏览器以固定 viewport 捕获原始截图。
7. 写入截图清单，计算哈希并执行敏感信息扫描。
8. 删除临时数据、日志、trace 和下载缓存，检查端口、Docker 和磁盘。
9. 在 README/Wiki 中引用已验证截图并运行文档门禁。

## 错误处理

- 数据源失败：保留错误记录，不改用合成数据；更换已支持的真实来源后重新记录来源和截止日。
- 股票数据不完整：选择另一只真实股票，或缩短为有完整证据的日期范围。
- 公式没有足够信号：在候选真实股票中选择信号更清晰者，不修改历史价格或回测规则。
- Worker/API 异常：停止拍摄并修复运行环境；正常截图不得隐藏异常状态。
- 截图含敏感信息：作废原图并重新拍摄；不能仅依赖模糊遮盖后继续使用。
- 中英文内容不一致：阻止 Wiki 发布，直到页面对、导航和步骤重新同步。
- GitHub Wiki 发布失败：保持主仓库 PR 未完成状态，不把本地草稿宣称为公开文档。

## 自动验收

### README 门禁

- GitHub 默认 `README.md` 为简体中文，且与 `README.en.md` 双向链接。
- 两份 README 章节、截图、安装资产名称和 Wiki 入口一致。
- README 不要求普通用户从源码构建，也不把 GitHub CLI 作为基本安装前置条件。
- README 保持规定的简洁范围，不重新承载完整使用手册。

### Wiki 门禁

- 功能索引中的每个功能 ID 都映射到存在的中文页、英文页、章节和截图。
- 每个页面有语言切换、共享导航、可执行步骤和至少一张有效截图。
- 所有内部链接和公开图片 URL 有效，不存在占位符。
- 中文 `Home.md` 是默认入口，英文首页和所有页面对可互相到达。
- 安装、响应式和多步骤流程满足各自的截图数量与状态要求。

### 真实截图门禁

- 行情相关截图清单包含合法 A 股代码、真实来源、截止日、复权和数据版本。
- 文件名、内容扫描和图注不得包含 `Synthetic` 或 `CC0 Demo`。
- 截图文件可解码、尺寸充足、非单色，SHA-256 与清单一致。
- 浏览器审阅确认图表不存在明显重复锯齿、组件重叠、裁切或错误状态。
- 对公开 Wiki URL 再执行一次语言、导航、截图和下载链接检查。

## 发布顺序

1. 在隔离分支完成 README、自动门禁、公开截图和截图清单。
2. 运行单元、文档、浏览器和公共树检查，并进行独立审阅。
3. 通过 Pull Request 合并主仓库变更到 `main`。
4. 推送 Wiki 中英文页面、导航和真实截图。
5. 对公开 README、Wiki 页面、图片和 Release 下载链接执行浏览器验收。
6. 在 v1 最终审计中记录主仓库 commit、Wiki commit、页面/截图矩阵和公开 URL。

现有 `v1.0.0` 安装包流水线继续完成。新增文档要求作为发布后文档收口交付，不移动已经发布的安装包标签，也不重写公开历史。

## 完成标准

- README 和 Wiki 默认中文且英文切换清晰。
- 23 对详细页面和功能索引公开可访问，所有已交付功能都有明确落点。
- 代表性行情、公式、回测和分析截图使用经清单验证的真实 A 股数据。
- Windows/macOS 安装教程和首次启动截图来自最终安装产物。
- 文档门禁、浏览器公开 URL 验收、敏感信息扫描和 final-release-audit 全部通过。
- 临时行情、数据库、浏览器 trace、日志和构建缓存已清理。
