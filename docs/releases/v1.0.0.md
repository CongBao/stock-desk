# stock-desk v1.0.0

`v1.0.0` is the first complete stock-desk release: a private, single-user,
low-code A-share research workstation for market charts, TDX-compatible
formulas, historical backtests, and evidence-linked multi-agent research.

## Installers and integrity

The GitHub Release publishes source-free installers for these targets:

- Windows x86_64
- macOS x86_64
- macOS arm64

- `stock-desk-1.0.0-windows-x86_64.exe`
- `stock-desk-1.0.0-macos-x86_64.dmg`
- `stock-desk-1.0.0-macos-arm64.dmg`

Each installer has a `.sha256` sidecar, a platform SPDX SBOM, and GitHub
artifact attestations. `SHA256SUMS.complete` covers the complete release asset
set, while `stock-desk.spdx.json` covers the source/runtime release bundle.
Follow the README verification commands before installing.

Windows code signing and macOS signing/notarization are applied only when the
corresponding release secrets are configured. The release page and clean-runner
records are the authority for the signing status of each published artifact.

## Included workflows

- Cached daily, weekly, and 60-minute A-share charts with adjustment, source
  provenance, pools, K-line main chart, and formula subcharts.
- A controlled TDX-compatible editor with templates, validation, preview,
  immutable versions, and BUY/SELL signals.
- Single-stock and pool backtests with next-open execution, T+1, suspension,
  price-limit, fee, tax, slippage, reproducibility, replay, and exports.
- DeepSeek-oriented, OpenAI-compatible, and Ollama research adapters with
  masked secrets, frozen run configuration, evidence links, retries, and
  insufficient-evidence behavior.
- A filterable task center plus responsive, keyboard-accessible icon navigation
  across desktop, narrow, tablet, short-landscape, and high-zoom layouts.

## Performance and release checks

The release enforces cached 2/3/5-second ceilings for chart interaction,
ten-year formula preview, and ten-year single-stock backtesting on the recorded
Ubuntu x64 4-core/16GB target. Full-A work remains asynchronous and must not
block the UI. Network time is excluded only when separately observed and
reported; the fixed performance workload forbids external requests.

The release pipeline also verifies requirements coverage, browser flows,
accessibility, secret redaction, formula isolation, dependency and container
security, migrations, backup/restore, rollback, public Git history, checksums,
SBOMs, provenance, and source-free first launch on each native target.

## Upgrade, backup, and rollback

Before upgrading, stop stock-desk and create a verified backup using the
documented backup tool. Native installations keep mutable data in the operating
system's per-user application-data directory. The v1 migration rehearsal covers
the published `v0.5.0` fixture and preserves user data across restart.

To roll back, stop the application, restore the backup made by the same
deployment identity, install the previous signed release, and verify health,
schema revision, data fingerprints, and task-worker startup. Never point older
code at a database already migrated by newer code without restoring its paired
backup.

## Scope and known limitations

- This is a local/private single-user tool, not a hosted multi-user service.
- It does not submit broker orders, automate live trading, or provide Level-2
  or tick-by-tick market feeds.
- Formula support is an explicitly documented TDX-compatible subset, not every
  TongdaXin function.
- Data and model providers require the user's own credentials, permissions, and
  network access. Provider availability, latency, and model output can change.
- Backtest win rate describes the selected historical sample and assumptions;
  it does not predict future performance.

stock-desk is for research assistance only and is not investment advice.

release evidence before v1 publication. The private specification authority was
checked against the implemented market, formula, backtest, analysis, packaging,
and publication scope without publishing local planning material. TradingAgents-CN
served as the primary capability and interaction reference; stock-desk's code,
data contracts, and UI implementation are independent.

Stage 5 preserves every predecessor gate and adds the integrated v1 checks.
`main` remains the canonical public default branch.
