# Stock Desk v1.1.0-beta.3

[English](#english)

## 中文

`v1.1.0-beta.3` 是仅面向 Windows x64 的未签名预发布版本。它保留 beta.2
的独立 Tauri 桌面窗口、普通用户安装、首次向导、股票搜索、响应式界面、公式与
回测体验，并加强候选安装器的可复现与供应链验证：

- 将 Tauri 2.11.4 使用的 NSIS 3.11 与 `nsis_tauri_utils` 0.5.3 绑定到经审计的
  官方下载摘要及完整提取树；
- 在受保护的私有快照内生成内容寻址的 NSIS 重打包工具包，拒绝绝对路径、未绑定
  文件、未知插件及可执行外部命令的编译期指令；
- 对同一 exact-SHA 候选执行两次独立重打包，要求字节完全一致并由严格回执绑定；
- main CI 通过后，Release 只复用不可变证明和候选，不重复运行源码测试或重新构建。

此版本没有 Authenticode 签名。Windows 可能显示“未知发布者”或 SmartScreen
提示；它不是稳定版或 Latest Release。SignPath Foundation 申请仍在等待审核，正式
`v1.1.0` 仍需要可信签名和 Windows 10/11 普通用户的真实 DPI/完整旅程验收。

本预发布不包含 macOS、Linux、Android 或 ARM64 资产，不连接券商、不执行交易，
也不会迁移或删除旧版 v1 数据。稳定使用请继续选择 `v1.0.0`。

## English

`v1.1.0-beta.3` is an unsigned Windows x64-only prerelease. It retains the
standalone desktop window hosting the same Web UI, current-user installation,
first-run setup, stock search, responsive UI, formulas, and backtesting
experience from beta.2 while tightening reproducibility and supply-chain
verification for the installer:

- NSIS 3.11 and `nsis_tauri_utils` 0.5.3 used by Tauri 2.11.4 are bound to
  audited official download digests and the complete extracted tree;
- a content-addressed NSIS repack kit is created inside protected private
  snapshots, rejecting absolute paths, unbound files, unknown plugins, and
  compile-time instructions capable of external execution;
- two independent repacks of the exact-SHA candidate must be byte-identical
  and are bound by strict receipts;
- after main CI succeeds, Release reuses the immutable proof and candidate
  without rerunning source tests or rebuilding the installer.

This version has no Authenticode signature. Windows may show an Unknown
Publisher or SmartScreen warning; it is neither stable nor Latest. The SignPath
Foundation application remains pending, and formal `v1.1.0` still requires
trusted signing plus real Windows 10/11 standard-user DPI and full-journey
acceptance.

This prerelease contains no macOS, Linux, Android, or ARM64 assets, does not
connect to a broker or place trades, and does not migrate or delete legacy v1
data. Continue to use `v1.0.0` for stable use.
